Internet and Web Technologies (CSCI 355)

Summer 2019

Term Project – Phase 1

Introduction:

The project for the Summer 2019 term is to build a “Search And Reporting Application” (SARA). The project will be implemented in two main phases. The first is to be completed individually. [The second phase, to be described in a separate document, will generally require working in groups. The recommended group size is 4 to 6 students. You should already be forming your groups now...

Overview:

  • Create a website which will be used to house SARA, as well as other pertinent information (see “Website/Homepage” below).
  • Display search results read from files stored in CSV, JSON, and XML formats.
  • Display search results obtained from using the Google Search API (see e.g. Google Api)

Regardless of the source of the search results (file, Google API, or custom search engine) – the presentation styling should be the same. This should all be controlled using an external stylesheet (CSS).

In addition, each displayed search result should have a checkbox next to it, and users should be able to save whichever search results they have checked to any of the three file types mentioned above. (The standard “Select All” and “Deselect All” options should be provided.) This write feature should be available across all sources of search results.

Website/Homepage:

The home page should have at least the following four high-level menu options.

  • Course, with a submenu items pointing to our courseZybook, TopHat, Google Drive, W3Schools.com, and optionally, other resourcesrelevant to our course.
  • Search, with submenu items to the various search modes specified in the Overview. This will obviously be the most important functional area of the site.
  • Browser, with five submenu items respectively communicating the following attributes.
    • Navigator* - appName, product, appVersion, userAgent, platform, language
    • Window - inherHeight, innerWidth
    • Screen - width, height, availWidth, availHeight, colorDepth, pixelDepth
    • Location - href, hostname, pathname, protocol
    • Geolocation - Latitude, Longitude

    See Java Script and scroll down to “JS Browser BOM” in left nav-bar to get ideas as well as relevant code snippets.

  • About - with submenu items to descriptions of the student-developers of the page and a “Contact Us” to contact them by e-mail. (If you don’t want your real email posted, you can create a Mailinator or other emailing alias system that will forward to your real email.)

The website should use “responsive web design” (RWD) – so that it can be easily viewed on mobile devices, as well as on larger screens such as laptops and desktops.
